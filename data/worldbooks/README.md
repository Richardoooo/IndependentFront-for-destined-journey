# 世界书填写指南

本文件告诉 AI 如何从参考材料（`reference/` 目录下的原版世界书）中提取内容，填入 `data/worldbooks/` 下的五个世界书文件。

---

## 一、世界书文件结构

每个 `.json` 文件：

```json
{
  "id": "唯一标识，与文件名一致",
  "name": "中文显示名",
  "partition": "分区类型标识",
  "description": "这个书给哪些 Agent 用，写什么内容",
  "entries": [
    {
      "id": "条目唯一 ID，用 kebab-case",
      "content": "注入给 AI 的正文，纯文本或 Markdown",
      "comment": "人类备注，不注入 AI",
      "enabled": true,
      "order": 100
    }
  ]
}
```

**字段说明：**

| 字段 | 用途 | 备注 |
|------|------|------|
| `id` | 条目唯一标识 | 建议格式：`分区名-主题`，如 `character_detail-human` |
| `content` | **注入给 AI 的正文** | 这是 AI 真正能看到的内容。纯文本或 Markdown，不要放 JSON 代码块。保持紧凑但信息密度高 |
| `comment` | 人类可读备注 | AI 看不到。用于标记来源（如原版世界书条目 #XXXXX） |
| `enabled` | 开关 | `true` = 注入，`false` = 暂时禁用 |
| `order` | 注入排序 | 数值越大越靠后（越接近 prompt 尾部 = 对 AI 影响越大）。默认 100 |

## 二、五个世界书的分工

### 1. `world_overview.json` — 世界总览
**注入给**: story, plot_pre_check, plot_post_check
**内容范围**:
- 世界观宇宙架构（虚海 / 神国 / 阿斯塔利亚位面）
- 纪元与历史（复兴纪元、位面入侵等大事件）
- 神明体系
- 魔法体系基础
- 10 势力总览（名字 + 一句话定位，详细地理放 region_detail）
- 种族总览（名字 + 一句话，详细放 character_detail）
- 世界主设定 (`#822383 [世界主设定]`)
- 叙事基调与风格

**不要放**: 数值公式、详细地名列表、单个角色介绍

---

### 2. `numerical_design.json` — 数值化设计
**注入给**: vars_update, craft_gen
**内容范围**:
- 核心数值表（世界书 `#417617 [核心数值表]`）HP/MP/SP 乘数、战斗系数、属性上限、EXP 上限
- 品质体系（7 级：普通/优良/稀有/史诗/传说/神话/唯一）
- 制作协议（`#683615 [生产制作协议]`）4 类型、3 级加工、DC 基准
- 战斗协议数值部分（`#837805 [战斗协议]`）伤害公式、命中评级、先攻公式
- 物品/装备品质规则（`#265160 [品质效果限定]`）
- 变量命名规范（`stat_data.*` 命名空间）
- FP 计算规则

**不要放**: 叙事描述、角色个例、地理信息

---

### 3. `character_detail.json` — 角色详细
**注入给**: char_update, char_gen, item_gen
**内容范围**:
- 23 种族详情（`#906373 [种族概览]` 及各种族条目）
- 血脉觉醒/继承规则
- 登神长阶体系（Lv.13+ 要素/权能/法则/神位/神国）
- 状态效果规则（层数/倒计时/来源/效果数值化格式）
- 角色生成规则（`#865613 [角色生成]`）
- 技能/装备/物品生成规格（`#261442` + `#265160`）
- 好感度体系（`#966681 [好感度]`）
- 死亡与复活规则

**不要放**: 单个 NPC 的详细传记（那是角色卡，不是世界书）

---

### 4. `region_detail.json` — 地区详细
**注入给**: story, plot_pre_check, plot_post_check
**内容范围**:
- 10 势力地理详情（各势力的 `[地理]` 条目）
- 首都与主要城市描述
- 地块/行省/公国详情
- 位置拓扑连通性
- 旅行规则（徒步/骑乘/马车/传送 日行距离）
- 气候与环境
- 冒险区域（无尽树海、龙脊山脉等）

**不要放**: 势力政治结构（放 world_overview）、单个 NPC 住所

---

### 5. `var_update.json` — 变量更新规则
**注入给**: vars_update
**内容范围**:
- VarsPatch 操作说明（replace / delta / insert / delta_time）
- 变量命名空间规范（`stat_data.世界.*` / `stat_data.主角.*` / `stat_data.任务列表.*` 等）
- 合法值域约束
- `$var` API 使用说明
- 变量路径格式举例
- 常见变量更新场景示例（位置变化、HP 变动、物品获得等）

**不要放**: 具体世界的变量值（那是运行时数据，不是规则）

## 三、条目导入规则

- **原版条目不再改动** — 直接整条搬入对应分区，不断句、不拆分、不改写
- **筛选归类** — 判断这条原版世界书属于哪个分区，放进对应的 `.json` 文件
- **content 原样保留** — 原版条目里是什么就放什么
- **comment 标注来源** — 格式：`来源: #XXXXXX [条目名]`

**示例**:

原版条目 `#822383 [世界主设定]`：
```
世界主设定:
  宇宙架构: 虚海/神国/阿斯塔利亚
  纪元: 复兴纪元
  ...
```

导入后：
```json
{
  "id": "world_overview-main-setting",
  "content": "世界主设定:\n  宇宙架构: 虚海/神国/阿斯塔利亚\n  纪元: 复兴纪元\n  ...",
  "comment": "来源: #822383 [世界主设定]",
  "enabled": true,
  "order": 100
}
```

如果一个条目同时涉及多个分区（如既讲数值又讲叙事），放入**最主要**的那个分区，不需要复制到多个文件。

---

## 四、来源标注

每个条目的 `comment` 字段标注来源，格式：

```
"comment": "来源: 世界书 #XXXXXX [条目名]"
```

如果一个条目合并了多个来源：

```
"comment": "来源: 世界书 #822383 [世界主设定] + #906373 [种族概览]"
```
